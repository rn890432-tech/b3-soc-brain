## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability:

1. Do not open a public issue.
Security issues should be reported privately to protect users.
2. Contact Maintainers Directly
Send details to: rn890432@gmail.com
3. Provide Details
• Steps to reproduce the vulnerability
• Potential impact
• Suggested fix (if known)

We will acknowledge receipt within 48 hours and provide updates on the investigation and resolution timeline.

---

## Disclosure Policy

• We aim to resolve confirmed vulnerabilities promptly.  
• Once fixed, we will release a patch and update documentation.  
• Public disclosure will occur only after users have had reasonable time to update.

---

## Best Practices for Contributors

• Always run tests before submitting code.  
• Avoid introducing dependencies with known vulnerabilities.  
• Use secure coding practices (e.g., input validation, avoiding hard‑coded secrets).  
• Never commit sensitive information (API keys, passwords, tokens).
